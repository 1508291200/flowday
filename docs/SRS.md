## 1. 引言

### 1.1 目的与范围

本文档旨在详细定义 **FlowDay** 日程管理工具的功能需求、非功能需求及用户交互规范。FlowDay 是一款以思维导图为核心组织逻辑的日程管理工具，通过三种互补的视图（思维导图、列表、筛选）满足用户从宏观规划到微观执行的全方位日程管理需求。

本文档的目标读者包括：
- 产品经理：理解产品需求，指导开发方向
- 开发工程师：获取详细技术规格，指导编码实现
- 测试工程师：编写测试用例，验证功能实现
- UI/UX 设计师：了解交互规范，设计用户界面

### 1.2 产品愿景

> **"让日程管理像思维导图一样自由延展，像列表一样清晰有序，像筛选器一样精准定位。"**

FlowDay 追求的核心价值：
- **统一性**：三种视图共享同一数据源，修改即时同步
- **灵活性**：支持从全局概览到细节查看的无缝切换
- **精确性**：多维度筛选系统帮助用户快速定位目标日程
- **扩展性**：模块化架构支持后续功能迭代和生态扩展

### 1.3 术语与定义

| 术语 | 定义 |
|------|------|
| 日程节点 (ScheduleNode) | 日程树中的基本单元，包含标题、描述、日期、标签等信息 |
| 父节点 / 子节点 | 树形结构中的上下级关系 |
| 折叠 / 展开 | 控制子节点是否显示的状态 |
| 重要度 (Importance) | 1-5 级评分，表示日程的重要程度 |
| 截止日期 (DueDate) | 日程需要完成的最后期限 |
| 标签 (Tag) | 用户自定义的分类标记 |
| 筛选器 (Filter) | 根据条件过滤日程的组件 |

## 2. 用户需求分析

### 2.1 目标用户画像

#### 用户类型 A：项目经理 (张明)
- **场景**：管理多个并行项目的里程碑和任务
- **痛点**：项目层级复杂，需要快速查看整体进度和特定阶段细节
- **期望**：通过思维导图查看项目全景，通过筛选快速定位延迟任务

#### 用户类型 B：产品经理 (李婷)
- **场景**：规划产品迭代，管理需求优先级
- **痛点**：需求之间存在依赖关系，需要灵活的层级管理
- **期望**：通过列表视图快速编辑，通过标签筛选特定类型需求

#### 用户类型 C：知识工作者 (王磊)
- **场景**：个人学习计划和目标管理
- **痛点**：年度目标需要拆解为月度、周度、日度任务
- **期望**：通过思维导图进行目标拆解，通过筛选查看特定时间段任务

### 2.2 用户故事

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-001 | 作为一个项目经理，我需要在思维导图中不断向外延展子任务 | 能创建多级子节点，支持无限层级 |

| US-005 | 作为一个用户，我需要为日程添加截止日期 | 能选择日期和时间，支持日期显示 |
| US-006 | 作为一个用户，我需要创建自定义标签来分类日程 | 能创建、编辑、删除标签，选择颜色 |
| US-007 | 作为一个项目经理，我需要筛选重要度大于4的紧急任务 | 筛选器能按重要度范围过滤 |

| US-010 | 作为一个用户，我需要标记日程完成状态 | 能通过复选框切换完成状态 |

## 3. 功能需求

### 3.1 三视图系统

#### 3.1.1 思维导图视图 (MindMap View)

**功能描述**：以树形结构横向展示日程，支持自由延展和折叠。

**核心功能**：

| 功能 | 描述 | 交互方式 |
|------|------|----------|
| F-M001 | 中心节点展示 | 默认显示根节点或用户选中的起始节点 |
| F-M002 | 子节点横向延展 | 子节点从父节点右侧水平展开 |
| F-M003 | 节点折叠 | 点击节点左侧折叠指示器，隐藏子节点 |
| F-M004 | 节点展开 | 点击折叠节点，恢复显示子节点 |
| F-M005 | 双击编辑标题 | 进入标题编辑模式 |
| F-M006 | 拖拽排序 | 拖拽节点可调整同级别顺序 |
| F-M007 | 拖拽改父子 | 拖拽到其他节点上可改变父子关系 |
| F-M008 | 添加子节点 | 快捷键或按钮添加子节点 |
| F-M009 | 删除节点 | 删除节点及其所有子节点 |
| F-M010 | 画布缩放 | 滚轮控制缩放比例 |
| F-M011 | 画布平移 | 空格键+鼠标拖拽平移画布 |
| F-M012 | 节点详情面板 | 点击节点显示详情编辑面板 |

**交互细节**：
- 首次加载时自动定位到根节点居中显示
- 节点间距：水平 180px，垂直 40px
- 折叠节点占据最小空间（24px 高度）
- 连接线使用贝塞尔曲线，自动绕过遮挡节点
- 支持键盘导航：方向键移动焦点，Enter 编辑，Delete 删除

**边界条件**：
- 单节点最大子节点数：无限制（虚拟滚动优化）
- 最大层级深度：无限制（建议超过 10 级时给出提示）
- 节点标题最大长度：200 字符

#### 3.1.2 列表视图 (List View)

**功能描述**：以大纲列表形式平铺展示日程，支持分级缩进和折叠。

**核心功能**：

| 功能 | 描述 | 交互方式 |
|------|------|----------|
| F-L001 | 树形列表渲染 | 自动根据父子关系渲染缩进 |
| F-L002 | 折叠/展开控制 | 点击行首折叠图标切换状态 |
| F-L003 | 快速添加同级 | Enter 键在下方添加同级节点 |
| F-L004 | 快速添加子级 | Tab 键添加为当前节点的子节点 |
| F-L005 | 拖拽调整层级 | 拖拽节点到目标位置改变父子关系 |
| F-L006 | 完成状态切换 | 点击复选框标记完成/未完成 |
| F-L007 | 标题快速编辑 | 双击标题进入编辑模式 |
| F-L008 | 批量选择 | Ctrl/Shift + 点击选择多个节点 |
| F-L009 | 批量操作 | 选中后可批量删除、移动、设置标签 |
| F-L010 | 展开/折叠全部 | 工具栏按钮一键展开/折叠所有节点 |

**交互细节**：
- 缩进单位：每级 24px
- 完成后节点显示删除线，颜色变淡
- 拖拽时显示插入位置指示线
- 支持多列显示：勾选框、标题、重要度、日期、标签

**边界条件**：
- 列表项最大数量：无限制（虚拟列表优化）
- 列表深度：无限制

#### 3.1.3 筛选视图 (Filter View)

**功能描述**：通过多维度筛选条件精确定位日程，以卡片列表展示结果。

**核心功能**：

| 功能 | 描述 | 交互方式 |
|------|------|----------|
| F-F001 | 重要度范围筛选 | 滑块选择重要度范围（如 3-5） |
| F-F002 | 日期范围筛选 | 日期选择器设置开始和结束日期 |
| F-F003 | 标签筛选 | 多选标签，包含任意/全部模式切换 |
| F-F004 | 完成状态筛选 | 全部/已完成/未完成三选一 |
| F-F005 | 关键字搜索 | 搜索标题和描述中的关键字 |
| F-F006 | 排序设置 | 选择排序字段和升序/降序 |
| F-F007 | 筛选结果展示 | 以卡片列表展示符合条件的节点 |
| F-F008 | 快速跳转 | 点击结果跳转到对应视图 |
| F-F009 | 保存筛选预设 | 将当前筛选条件保存为预设 |
| F-F010 | 加载筛选预设 | 从预设列表中选择并应用 |

**筛选条件组合**：
```
最终筛选结果 = 重要度符合 AND 日期符合 AND 标签符合 AND 状态符合 AND 关键字匹配
```

**排序选项**：
| 排序字段 | 说明 |
|----------|------|
| importance | 按重要度排序（高→低或低→高） |
| dueDate | 按截止日期排序（近→远或远→近） |
| createdAt | 按创建时间排序 |
| title | 按标题字母顺序排序 |

**预设保存格式**：
```json
{
  "name": "本周紧急任务",
  "config": {
    "importanceRange": [4, 5],
    "dueDateRange": { "start": "2026-04-28", "end": "2026-05-04" },
    "tags": [],
    "completed": false,
    "sortBy": "importance",
    "sortOrder": "desc"
  }
}
```

### 3.2 属性管理系统

#### 3.2.1 日程属性

每个日程节点具有以下属性：

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| id | string (UUID) | 是 | 自动生成 | 全局唯一标识 |

| importance | number (1-5) | 是 | 3 | 重要程度 |
| dueDate | string (ISO8601) | 否 | null | 截止日期时间 |
| tags | string[] | 是 | [] | 标签ID数组 |
| collapsed | boolean | 是 | false | 是否折叠 |
| order | number | 是 | 0 | 同级排序权重 |
| completed | boolean | 是 | false | 是否完成 |
| createdAt | string (ISO8601) | 是 | 自动生成 | 创建时间 |
| updatedAt | string (ISO8601) | 是 | 自动生成 | 更新时间 |

### 3.3 数据同步机制

#### 3.3.1 发布-订阅模式

```
                    ┌─────────────────┐
                    │   Core Data     │
                    │     Layer       │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        ┌──────────┐  ┌──────────┐  ┌──────────┐
        │  Event   │  │  Event   │  │  Event   │
        │  Bus     │  │  Bus     │  │  Bus     │
        └────┬─────┘  └────┬─────┘  └────┬─────┘
             │             │             │
             ▼             ▼             ▼
      ┌───────────┐ ┌───────────┐ ┌───────────┐
      │  MindMap  │ │   List    │ │  Filter   │
      │   View    │ │   View    │ │   View    │
      └───────────┘ └───────────┘ └───────────┘
```

**事件类型**：
| 事件名 | 触发时机 | 携带数据 |
|--------|----------|----------|
| `node:created` | 新建节点时 | 节点数据 |
| `node:updated` | 节点属性变更时 | 节点数据 |
| `node:deleted` | 删除节点时 | 节点ID |
| `node:moved` | 节点移动位置时 | 节点ID、新父ID、新序号 |
| `node:expanded` | 节点展开时 | 节点ID |
| `node:collapsed` | 节点折叠时 | 节点ID |
| `tag:created` | 创建标签时 | 标签数据 |
| `tag:updated` | 更新标签时 | 标签数据 |
| `tag:deleted` | 删除标签时 | 标签ID |
| `filter:changed` | 筛选条件变更时 | 筛选配置 |
| `tree:reordered` | 整体树结构重排时 | 根节点 |

#### 3.3.2 数据一致性保证

1. **原子操作**：所有数据修改通过 Core 层统一处理
2. **事务支持**：批量操作（如删除带子节点）支持事务回滚
3. **乐观更新**：视图先显示更新，后台验证数据合法性
4. **冲突检测**：若检测到数据版本冲突，提示用户确认

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 测试场景 |
|------|------|----------|
| 首屏加载时间 | ≤ 2 秒 | 首次打开应用 |
| 视图切换时间 | ≤ 100ms | 思维导图↔列表↔筛选 |
| 数据同步延迟 | ≤ 50ms | 任一视图修改到其他视图更新 |
| 节点渲染性能 | 1000 节点内流畅 | 思维导图展开 1000 节点 |
| 筛选响应时间 | ≤ 200ms | 筛选条件变更到结果展示 |
| 本地存储速度 | ≤ 100ms | 单次数据保存 |

**优化策略**：
- 虚拟滚动：列表和筛选结果使用虚拟列表
- 懒加载：思维导图仅渲染可见区域节点
- 增量渲染：数据更新时仅重绘受影响部分
- 索引优化：标签、重要度等字段建立索引

### 4.2 兼容性需求

| 类型 | 要求 |
|------|------|
| 浏览器支持 | Chrome 90+, Firefox 88+, Safari 14+, Edge 90+ |
| 屏幕适配 | 支持 1280x720 及以上分辨率 |
| 响应式布局 | 支持窗口大小调整，适配不同屏幕 |
| 键盘操作 | 支持完整键盘导航（Tab, Arrow, Enter, Esc） |

### 4.3 可用性需求

| 指标 | 要求 |
|------|------|
| 学习成本 | 基础功能 10 分钟内可掌握 |
| 操作反馈 | 所有操作应有即时视觉反馈 |
| 错误提示 | 操作失败时提供明确错误信息 |
| 撤销支持 | 支持撤销最近 20 次操作 |
| 快捷键覆盖 | 常用操作均有快捷键 |

### 4.4 可维护性需求

| 需求 | 说明 |
|------|------|
| 模块化架构 | 各模块独立开发、测试、部署 |
| 接口契约 | 模块间通过明确接口通信 |
| 配置驱动 | 视图样式、行为通过配置而非硬编码 |
| 日志系统 | 记录关键操作和错误日志 |
| 监控指标 | 性能监控、错误监控、使用统计 |

## 7. 验收标准

### 7.1 功能验收

| 模块 | 验收条件 | 测试方法 |
|------|----------|----------|
| 思维导图 | 能创建、编辑、删除任意层级节点 | 操作节点，验证数据变化 |
| 思维导图 | 折叠展开正确隐藏/显示子节点 | 操作折叠，验证子节点状态 |
| 思维导图 | 画布缩放平移流畅 | 操作缩放平移，验证性能 |
| 列表视图 | 层级缩进正确显示 | 对比思维导图，验证结构一致 |
| 列表视图 | 拖拽能改变父子关系 | 拖拽节点，验证树结构变化 |
| 筛选视图 | 所有筛选条件正确过滤数据 | 分别测试各条件，验证结果 |
| 筛选视图 | 排序功能正确工作 | 测试各排序字段和方向 |
| 数据同步 | 任意视图修改，其他视图即时更新 | 操作任一视图，验证同步 |
| 标签系统 | 能创建、编辑、删除标签 | 操作标签，验证功能 |
| 属性编辑 | 重要度、日期等属性正确保存 | 修改属性，验证持久化 |

### 7.2 性能验收

| 指标 | 验收条件 |
|------|----------|
| 首屏加载 | ≤ 2 秒显示主界面 |
| 视图切换 | ≤ 100ms 完成切换动画 |
| 100节点渲染 | 流畅无卡顿 |
| 1000节点渲染 | 基本可用（可能轻微卡顿） |
| 数据保存 | ≤ 100ms 完成单次保存 |

### 7.3 稳定性验收

| 场景 | 验收条件 |

| 刷新恢复 | 刷新页面后数据正确恢复 |
